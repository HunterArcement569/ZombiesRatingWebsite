The intention of the index.html is to describe the rating system proposed, and act as a central navigation hub for the user.

The intention of the RatingPage.html is to perform three different options, generated based on query parameters labeled as functions A, B, and C.
-A will create a rating
-B will edit a rating (imported via the *.json model)
-C will view a rating (also imported via the *.json model)
-All ratings will allow for exportation of rating model via *.json
-Most of the ui on the RatingPage.html will be generated from the .js file since it's based on what function it being executed

The actual rating system is as follows:
-User first rates Call of Duty Zombies Games, the data and model of which are stored in the GameData.js file
-User then rates the Zombies maps, the data and model of which are stored in the MapData.js file
-Each set of ratings are stored in the RatingData object which is what will be serialized/deserialized to/from json
-It is intended that as new games and maps are added to this system that old exported ratings will still work
-A map's final rating will be calculated via:
    (Map Total Points/Map Maximum Points) * (Game Total Points/Game Maximum Points) * 100
-The display will show them as a tier-list (ordered top to bottom and left to right in each tier), each tier is denoted below:
    A -> 90 - 100
    B -> 80 - 89
    C -> 70 -> 79
    D -> 60 -> 69
    E -> 50 -> 59
    F -> 0 -> 49

-Users can select different filters for their ratings, so if they want to do only Treyarch games for example, then they'll only rate Treyarch games and their related maps
-Once a user has finished rating their maps they will be sent to the rating viewer (which shows that final tier list) and is where they can also export that rating as needed

-Animations for buttons should follow a similar style to that of the home page
